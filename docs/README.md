# Docs Library

This repository now uses the harness-oriented numbered `docs/` layout for all new planning, governance, regression, and walkthrough work.

- `docs/01-GOVERNANCE/`
- `docs/03-ARCHITECTURE/`
- `docs/04-DEVELOPMENT/`
- `docs/05-TEST-QA/`
- `docs/06-INTEGRATIONS/`
- `docs/07-OPERATIONS/`
- `docs/08-SECURITY/`
- `docs/09-PLANNING/`
- `docs/10-WALKTHROUGH/`
- `docs/11-REFERENCE/`
- `docs/99-TMP/`

## Legacy Directories

The following legacy paths are preserved for historical continuity and should not be deleted during bootstrap:

- `docs/plans/`
- `docs/tasks/`
- `docs/archive/`

New non-trivial work should use the numbered harness layout instead of adding more files under the legacy paths.
