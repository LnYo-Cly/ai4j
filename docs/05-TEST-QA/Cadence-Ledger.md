# Cadence Ledger - ai4j-sdk

> Last updated: 2026-06-09
> Defines which regression gates should be revisited when each repository surface changes.

## Cadence Terms

| Cadence | Meaning | Default Evidence |
|---------|---------|------------------|
| touched-surface | run before closeout when the task edits the owning module or surface | cited command output in task `progress.md` |
| PR | run through CI when a pull request touches the configured paths | workflow run or queued/pending workflow evidence |
| merge-batch | refresh the full local baseline for integration to `dev` or `main` | SRB row plus command/workflow evidence |
| opt-in-live | run only when the task or release explicitly needs real provider, hosted demo, signing, or publishing behavior | sanitized L3-L5 evidence plus env var names, never secret values |
