# Cadence Ledger - ai4j-sdk

> Last updated: 2026-04-28
> Defines which regression gates should be revisited when each repository surface changes.

## Trigger Rules

| Change Scope | Triggered Regression Gates | Why |
|--------------|----------------------------|-----|
| PR to `dev` or `main` touching `pom.xml`, `ai4j/**`, `ai4j-agent/**`, `ai4j-coding/**`, `ai4j-cli/**`, `ai4j-spring-boot-starter/**`, `ai4j-flowgram-spring-boot-starter/**`, or `ai4j-bom/**` | RG-001, RG-002, RG-003, RG-004, RG-005, RG-006, RG-007 | automatically triggered by `.github/workflows/java-regression.yml` |
| `ai4j/` provider, protocol, RAG, vector, MCP, image, audio, realtime, or agentflow connector changes | RG-001, RG-007 | core SDK contract plus cross-module packaging |
| `ai4j-agent/` workflow, memory, trace, or orchestration changes | RG-002, RG-007 | runtime behavior plus dependency alignment |
| `ai4j-coding/` tools, outer-loop, checkpoint, or shell/apply-patch changes | RG-003, RG-007 | coding runtime plus cross-module packaging |
| `ai4j-cli/` CLI, TUI, ACP, session, or rendering changes | RG-004, RG-007 | host behavior plus packaged artifact alignment |
| `ai4j-spring-boot-starter/` property binding or auto-configuration changes | RG-005, RG-007 | starter wiring plus repo-wide packaging |
| `ai4j-flowgram-spring-boot-starter/` or `ai4j-flowgram-demo/` changes | RG-006, RG-007, RG-009 | backend FlowGram surface, packaging, and likely web-demo contract impact |
| `docs-site/` content, config, or workflow changes | RG-008 | docs-site build is the owning gate |
| `ai4j-flowgram-webapp-demo/` frontend changes | RG-009 | frontend demo build/lint/type surface |
| root `pom.xml`, `ai4j-bom/`, shared build plugins, release/publishing logic | RG-001, RG-002, RG-003, RG-004, RG-005, RG-006, RG-007 | shared dependency or packaging changes can break all Java surfaces |
| `AGENTS.md`, `docs/11-REFERENCE/`, SSoT, or workflow governance changes | review affected gates manually; keep existing gate map unless executable behavior changes | governance docs alone do not automatically imply code regression |
| any merge to `dev` or `main` | Full Shared Batch (`RG-001` to `RG-009`) | branch integration should refresh the full control surface |
