# docs-site hard technical tone polish - Findings

## Discoveries

### Residual issues are tonal, not structural

- Why it mattered: the main docs-site cleanup waves already resolved the larger content, structure, and route problems
- What was found: only a small set of canonical pages still contained subjective wording such as `最稳定`, `最稳妥`, `最通用`, or `更合适的起点`
- Impact on plan: treat this as a narrow tone-polish wave rather than another broad documentation rewrite

## Decisions

| Decision | Choice | Reason | Alternatives |
|----------|--------|--------|--------------|
| Scope size | keep this as a tiny finishing wave | the remaining issues are wording-only and tightly bounded | reopen a larger docs-site cleanup stream |
| Target pages | edit only canonical pages, not placeholder migration pages | the user concern is technical-doc tone, not legacy-link mechanics | rewrite placeholders that are intentionally short |
