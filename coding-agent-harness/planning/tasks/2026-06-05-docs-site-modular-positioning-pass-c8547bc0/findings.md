# docs site modular positioning pass - 发现记录

本文件记录任务执行中形成的判断、事实和技术决策。它不是审查报告；阻塞性问题请写入 `review.md`。

## 研究发现

### 模块独立性已有工程事实

- 背景：用户质疑“目前模块难道不够独立吗”，需要从仓库事实回答。
- 发现：根 `pom.xml` 已声明 `ai4j`、`ai4j-agent`、`ai4j-coding`、`ai4j-cli`、Spring Boot starter、FlowGram starter/demo 和 BOM；模块 POM 显示依赖从 `ai4j` 向 starter、agent、coding、cli、flowgram 逐层叠加。
- 影响：文案不应写成“未来要做模块独立”，而应写成“已有模块可以按需取用，并按阶段升级”。
- 后续：如需证明每个 artifact 的可发布独立性，应另开 Maven 依赖审计任务。

### docs-site 需要把目录事实翻译成用户路径

- 背景：前一轮入口页已经有模块地图，但仍偏“仓库结构说明”。
- 发现：用户真正关心的是“我只想解决当前问题，应该引入哪一块”，而不是模块有几个。
- 影响：首页增加 `用多少，取多少`，Why AI4J 增加渐进升级章节，Feature Map 增加 `按模块取用` 表。
- 后续：后续每个深页可继续补“最小依赖 / 升级到下一层 / 不适合场景”。
