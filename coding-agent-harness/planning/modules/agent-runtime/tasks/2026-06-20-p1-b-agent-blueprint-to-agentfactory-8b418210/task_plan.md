# P1-B Agent Blueprint to AgentFactory

Task Package Index: required

## 目标

在 `ai4j-agent` 中新增最小可用 `AgentFactory`，把 P1-A 的单 Agent Blueprint DTO 转换为真实 `AgentBuilder` / `Agent`，让 YAML Blueprint 从“可加载/校验”推进到“可由宿主提供依赖后创建 Agent”。

## 范围

- 做什么：新增 Factory / Context / Exception / resolver 或 options 类型；映射 model name、model options、instructions、workflow mode、workflow maxTurns；提供 deterministic tests；更新 docs-site Agent Blueprint 页面和 roadmap。
- 不做什么：不读取 provider token；不从本机 profile 创建 provider client；不安装插件；不扫描插件目录；不创建真实 sandbox；不做 CLI `ai4j run agent.yaml`；不做 Team/Workflow graph DSL。
- 主要风险：Factory 过早绑定 provider/profile/plugin/sandbox；Blueprint 字段被静默忽略导致用户误解；测试误触 live provider；破坏 Java 8 兼容。

## 预算选择

选择预算：complex

选择理由：该任务会形成 P1-B 公共 API，影响后续 CLI/Runner/FlowGram 消费 Blueprint 的入口，需要明确 host-supplied resolver 边界、异常语义、文档和回归证据。

## 设计方案

采用 host-supplied context/resolver 方案：

```text
AgentBlueprint -> AgentFactory -> AgentFactoryContext/resolvers -> AgentBuilder -> Agent
```

Factory 只做确定性映射，不拥有 secret、不连接外部 provider、不安装插件、不创建 sandbox。
